# ICMDA 2026 숙소배정 대시보드 — 사용법

`room-dashboard.html` — 더블클릭하면 브라우저에서 열리는 **단일 파일** 대시보드. (인터넷 필요: SheetJS CDN 사용)

## 흐름
1. **① 마스터 열기** → 최신 `ICMDA Master Data v##.xlsx` 선택. (`참가자`·`객실` 탭을 헤더 이름으로 읽으므로 열 순서를 바꿔도 됨)
2. **② 자동 배정** → 제약을 지키며 자동 배치:
   - `배정필요=No` 제외, `배정숙소`에 **부영호텔**이 있으면 외부 예약으로 **고정(잠금)**.
   - **가족(여러 명 한 행)**은 방을 통째로 단독 사용. **개인**은 방의 남/여 지정에 맞춰 같은 성별끼리만.
   - 팀(자원봉사/찬양/미디어)·가족 묶음(그룹ID)은 같은 방으로 모으려 시도, 온돌방 우선 사용.
3. **수동 조정** → 사람 카드를 다른 방/미배정함으로 **드래그**. 🔓/🔒로 잠그면 다음 자동배정에서 제외.
4. **저장**
   - **Excel 저장** → `ICMDA 숙소배정 결과.xlsx` (`배정결과` + `객실roster` 2탭).
   - **진행 저장(JSON)** → 나중에 **진행 불러오기**로 이어서 편집. (마스터를 먼저 열고 JSON을 불러올 것)
5. **인쇄 / PDF** → 브라우저 인쇄 → PDF로 저장. 숙소별로 페이지가 나뉘고(부영수련원=동/층, 유어스=객실유형) **동일한 표 레이아웃**.

## 보기 전환 (오른쪽 위 탭)
- **배정 보기** — 숙소→객실→사람 칩 (드래그/배정).
- **타임라인 보기** — 참가자별 숙박 기간을 막대로 표시. 맨 왼쪽에 **ID 열**(참가자ID) 표시.
  - **숙소별로 구분**(부영수련원/유어스/부영호텔/미배정), 색 띠를 **클릭하면 접기/펼치기**(▾/▸). 행은 컴팩트.
  - 막대 **양끝을 끌면** 숙박기간(도착/출발) 조정, **가운데를 끌면** 통째로 이동. **막대 더블클릭=상세 카드**.
  - 숙박이 없는 사람은 트랙을 **클릭**하면 1박이 생기고, 이후 끌어서 조정.
  - **이름 칸을 끌어 위/아래로 순서 변경**. 다른 숙소 그룹의 행 위로 끌어다 놓으면 그 숙소로 **재배정**(가능한 방 자동 선택).
  - 6/26~6/28(공식지원 외)=분홍, 7/4 이후 **여유(연장) 칸**=회색 — 더 오래 머무는 사람은 막대를 오른쪽 칸으로 끌어 연장.
  - 검색·숙소·카테고리 필터로 좁혀 보기(최대 500명 표시).

## 상세 카드
- 사람을 **더블클릭**(칩·타임라인 이름·막대 어디서나)하면 상세 카드(인적사항·숙박·인원/동반·참가행사·현재 배정)가 뜸. 빈 항목은 생략.

## 색·표시
- 칩 왼쪽 띠: 파랑=남, 분홍=여, 회색=미상(?), 초록=가족(MIX).
- **카테고리 뱃지(색+약칭)**: 개인/가족/NOC/AFF/중국/자원봉사/미디어/찬양 — 칩·타임라인·카드에 색으로 구분.
- **👪N** = 동반 포함 총인원 N명(총인원 기준). 방 우측 `사용/정원` 뱃지: 초록=여유, 황금=만실, 빨강=초과.
- 빨간 테두리 칩 = 성별/정원 충돌(아래 경고 패널에 사유).
- **자동 배정** 클릭 시 처리 중 로딩 표시.

## 알아둘 점
- **부영호텔 ≠ 부영청소년수련원.** 부영호텔(부영 리조트)은 NOC 29명이 개별 예약한 **제3 숙소**로, 객실 인벤토리에 없음 → 대시보드에서 "개별 예약"으로 고정 표시.
- **정원이 빠듯함.** 인벤토리 수요 ≈718 vs 정원 714. 자동배정 후 **개인 남성 약 17명이 미배정**으로 남는 게 정상(실제 부족분). 해결: 온돌 117–123(가정 7실) 확정, 부영호텔 추가 객실, 또는 인원 조정.
- 마스터를 수정하면 다시 **① 마스터 열기**로 새로 불러와야 반영됨(라이브 연결 아님).
